# NOS Computers — Website

A modern, dark-themed, single-page marketing site for **NOS Computers**, a computer sales, service, and repair shop in Tucson, AZ.

🔗 **Live site:** https://yuenvision.github.io/nos-computer/
🔗 **Reference / original site:** https://noscomputerstucson.com/

The site is built with **[Eleventy](https://www.11ty.dev/)** from editable content files and deploys as **fully static HTML** to GitHub Pages. Content (prices, hours, copy, etc.) is edited through a friendly web form via **[Pages CMS](https://pagescms.org)** — no code required.

---

## Highlights

- **Single long-form page** with smooth-scroll navigation (Services · PC Builds · Gallery · Warranty · Contact).
- **Dark, modern aesthetic** — electric-cyan primary accent with a warm amber secondary (a nod to the NOS chrome/flame logo).
- **Service detail modals** — each service card opens a modal with full descriptions and pricing.
- **PC Builds carousel** — Entry / Mid / Turbo / Max tiers with full specs, on real build photos.
- **Gallery lightbox** — click any photo for a full-size view with keyboard (←/→) and on-screen navigation.
- **Warranty breakdown**, embedded **YouTube intro video**, and a **Google Maps** embed in the contact section.
- **Accessibility-minded for an older client base** — larger base font, high contrast, big tap targets, prominent click-to-call.
- **Fully responsive**, with a mobile menu. Scroll-reveal animations are a progressive enhancement (content is fully visible without JS).
- **Content-managed** — text/prices/contact info live in data files, editable through a web form. Output is still 100% static (great for SEO and speed).

---

## Editing the content (non-developers)

You don't touch code to update the site. See **[EDITING.md](EDITING.md)** for the full walkthrough. In short: open **[app.pagescms.org](https://app.pagescms.org)**, sign in, edit a screen (⭐ Prices, ⭐ Business Info, etc.), and **Save** — the site rebuilds and goes live in a minute or two.

First-time technical setup (installing the CMS app, deploy config) is documented in **[CMS-SETUP.md](CMS-SETUP.md)**.

## Project structure

```
.
├── src/
│   ├── index.html              # The page — an Eleventy (Nunjucks) template
│   ├── _data/                  # ← editable content (what the CMS edits)
│   │   ├── business.yml        #   phone, email, address, hours, social
│   │   ├── prices.yml          #   service/build prices
│   │   ├── services.yml        #   service cards + modal copy
│   │   ├── builds.yml          #   the 4 PC build tiers + specs
│   │   ├── hero.yml            #   hero headline + intro
│   │   ├── warranty.yml        #   warranty tiers + banner
│   │   └── site.yml            #   page title, SEO text, YouTube ID, footer
│   └── assets/
│       ├── css/styles.css      # All styling (theme tokens in :root at top)
│       ├── js/main.js          # Nav, modals, gallery lightbox, builds carousel, reveal
│       └── img/ , video/       # Logo, service/gallery photos, hero video
├── .eleventy.js                # Build config (input src/ → output _site/)
├── .pages.yml                  # Pages CMS config — defines the editing screens
├── .github/workflows/deploy.yml# Build + deploy to GitHub Pages on push to master
├── package.json                # Eleventy + js-yaml (build-time only)
├── scripts/smoke-test.sh       # Build + serve + sanity checks
├── CMS-SETUP.md                # One-time CMS / deploy setup
├── EDITING.md                  # How Shawn edits content
└── README.md
```

The build output (`_site/`) and `node_modules/` are git-ignored; they're regenerated by the build.

## Running locally

Requires Node.js (18+).

```bash
npm install          # once
npm start            # build + live-reload server at http://localhost:8765/
# or:
npm run build        # one-off build into _site/
bash scripts/smoke-test.sh   # build, serve, and run sanity checks
```

---

## Deployment

Pushing to **`master`** triggers `.github/workflows/deploy.yml`, which builds the site with Eleventy and publishes `_site/` to GitHub Pages (~1–2 min). **Editing in Pages CMS commits to `master`, so a Save publishes automatically.** If a build fails, the last good deploy stays live.

> Future option: move hosting to DreamHost (the build/CMS stay the same; only the deploy target changes). Not set up yet.

## Editing quick reference

| To change… | Edit… | How |
|---|---|---|
| Prices, hours, phone/email/address, social links | `src/_data/business.yml`, `prices.yml` | **Pages CMS** (⭐ screens) |
| Service / build / hero / warranty copy, SEO, YouTube ID | the matching `src/_data/*.yml` | **Pages CMS** |
| Theme colors / spacing | `:root` in `src/assets/css/styles.css` | code |
| Layout, section order, gallery photos, card images, hero video | `src/index.html`, `src/assets/` | code |

Anything in `src/_data/` is also editable directly in the CMS; everything else is a developer change.
